# NERForge — Backend

FastAPI-сервис: генерация обучающего датасета, обучение spaCy NER-модели и разбор
текста. **Без базы данных и брокеров** — статус обучения хранится в памяти, само
обучение идёт фоновой `asyncio`-задачей (тяжёлый `spacy train` уходит в дочерний
процесс, поэтому сервер не блокируется). На диске хранится только **последний**
результат — ничего не накапливается.

## Стек

Python 3.14, FastAPI, spaCy, pandas, sse-starlette, loguru.

## API (префикс `/v1`)

| Метод | Путь | Назначение |
|-------|------|-----------|
| POST | `/generate` | Принять эталонный CSV + параметры шума → вернуть зашумлённый обучающий CSV (синхронно). |
| POST | `/train` | Принять обучающий CSV + параметры → запустить обучение в фоне, вернуть `job_id`. |
| GET | `/train/{id}` | Текущий статус/прогресс/метрики задачи. |
| GET | `/train/{id}/stream` | SSE-поток прогресса до завершения. |
| GET | `/train/{id}/model` | Скачать zip обученной модели (после успеха). |
| POST | `/parse` | Принять zip модели + текст → вернуть разбивку по полям. |

Интерактивная документация: `http://localhost:8000/docs`.

## Запуск

### Docker (из корня репозитория)

```bash
docker compose up --build        # поднимет backend + frontend
```

> Образ по умолчанию скачивает `sm` + `md` (тяжёлая `lg` — нет). Список задаётся
> build-arg `SPACY_MODELS` (в корневом `docker-compose.yml` или при ручной сборке):
> ```bash
> docker build --build-arg SPACY_MODELS="ru_core_news_sm ru_core_news_md ru_core_news_lg" \
>   -t nerforge-backend ./backend
> ```
> Какие модели реально установлены — отдаёт `GET /v1/train/base-models`. Попытка
> обучать на неустановленной модели вернёт 409 с инструкцией (фронт это предупреждает заранее).

### Локально

```bash
python3.14 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python -m spacy download ru_core_news_sm   # при желании также md и lg
uvicorn src.main:app --reload --port 8000
```

В VS Code есть готовый конфиг отладки — `.vscode/launch.json` (открой папку `backend`).

## Переменные окружения (`.env`)

| Переменная | По умолчанию | Описание |
|-----------|--------------|----------|
| `NERFORGE_DATA_DIR` | `./data` | Куда складывать рабочие папки и загруженные модели. |
| `NERFORGE_CORS_ORIGINS` | `["http://localhost:3000"]` | Разрешённые origin'ы фронтенда (JSON-массив). |

Скопируй `.env.example` в `.env` при необходимости (для дефолтов не обязательно).

## Как устроен код

```
src/
├── main.py                     # сборка приложения, CORS, роутеры
├── core/                       # config, logger, storage (пути + очистка), exceptions
├── features/
│   ├── generation/             # шаг 1: генерация CSV (синхронно)
│   └── inference/              # шаг 3: парсинг текста загруженной моделью
└── use_cases/
    └── training/               # шаг 2: обучение
        ├── augmentation.py     # шум + сборка обучающих примеров (корректные спаны)
        ├── trainer.py          # обёртка над `spacy train` + упаковка модели в zip
        ├── store.py            # in-memory статус (только текущая задача)
        ├── service.py          # фоновый прогон обучения
        └── router.py           # HTTP + SSE + скачивание
```

### Логика обучения вкратце

1. CSV (колонки = метки) → строки `{МЕТКА: значение}`.
2. `augmentation.build_training_samples` собирает сырые строки: перемешивает порядок
   полей и считает символьные спаны **курсором** (не через `find`, поэтому коллизии
   значений вроде `10мг`/`10 шт` не ломают разметку).
3. spaCy обучается (`sm` → без статических векторов; `md`/`lg` → с их векторами).
4. `model-best` упаковывается в zip и отдаётся на скачивание.
