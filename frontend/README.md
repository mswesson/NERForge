# NERForge — Frontend

Веб-интерфейс NERForge: три экрана-шага, общающиеся с бэкендом по HTTP/SSE.

## Стек

React 19, TypeScript, Vite, Tailwind CSS, lucide-react (иконки).

## Три экрана

1. **Конструктор** (`DatasetGenerator`) — загрузка эталонного CSV, настройка шума
   (объём выборки, доля шума, опечатки, сжатие пробелов, сокращения, нижний регистр),
   генерация и скачивание обучающего CSV.
2. **Обучение** (`ModelTrainer`) — загрузка обучающего CSV, выбор базовой модели
   (sm/md/lg), эпох и dropout; запуск обучения, прогресс и метрики в реальном времени
   (SSE), скачивание готовой модели (zip).
3. **Тестирование** (`PredictionPlayground`) — загрузка файла модели, ввод примеров,
   подсветка распознанных сущностей и таблица извлечённых полей.

Весь обмен с сервером — в `src/api/client.ts`.

## Запуск

### Docker (из корня репозитория)

```bash
docker compose up --build        # поднимет frontend + backend
```

### Локально

```bash
cp .env.example .env             # VITE_API_URL = http://localhost:8000/v1
npm install
npm run dev                      # http://localhost:3000
```

Нужен запущенный бэкенд (по умолчанию на `http://localhost:8000`).

## Переменные окружения (`.env`)

| Переменная | По умолчанию | Описание |
|-----------|--------------|----------|
| `VITE_API_URL` | `http://localhost:8000/v1` | Базовый URL API бэкенда. |

> Vite зашивает `VITE_*` на этапе сборки. В Docker значение передаётся build-аргументом
> `VITE_API_URL` (см. корневой `docker-compose.yml`).

## Скрипты

| Команда | Что делает |
|---------|-----------|
| `npm run dev` | Dev-сервер с HMR на порту 3000. |
| `npm run build` | Продакшен-сборка в `dist/`. |
| `npm run preview` | Локальный предпросмотр собранного `dist/`. |
| `npm run lint` | Проверка типов (`tsc --noEmit`). |
